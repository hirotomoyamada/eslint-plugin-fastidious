# Branch Rules

**Format:** `<type>/<description>`

- `type` matches the [Commit Rules](commit.md) types (`feat`, `fix`, `test`, `docs`, `refactor`, `chore`, `ci`, `build`, `perf`, `style`).
- `description` is kebab-case and briefly identifies the scope or area (rule, config, workflow, docs, or topic).

**Examples:**

```text
feat/add-no-abbreviated-variable-rule
fix/no-single-letter-variable-destructuring
test/no-single-letter-variable-cases
docs/rule-configuration
build/package-output
```
